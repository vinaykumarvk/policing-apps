## 2. Accessibility (WCAG)

### PASS -- Strong Foundations
- **Skip links:** Present on every view across all 6 apps, properly hidden and visible on focus.
- **Focus management:** Global `*:focus-visible` with `3px solid var(--color-focus)` (desktop), reduced to `2px` on mobile.
- **ARIA attributes:** 77 total `aria-*` usages in citizen app alone; `aria-expanded` on all expandable controls (avatar menus, accordion sections).
- **Modal a11y:** `role="dialog"`, `aria-modal="true"`, `aria-labelledby`, Escape dismiss, focus trap, focus restoration on close.
- **Drawer a11y:** Portal-based, sets `inert` on `#root`, focus trap with Tab cycling, Escape close, auto-focus first focusable element.
- **Toast a11y:** `aria-live="polite"`, `role="status"`, dismiss button has `aria-label`.
- **Breadcrumb:** Semantic `<nav aria-label="Breadcrumb">`, `aria-current="page"`.
- **Progressbar:** `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, `aria-label`.
- **Form fields:** `FieldContext` auto-wires `aria-describedby` (hint + error IDs), `aria-invalid` on error state.
- **Error alerts:** `role="alert"` on error messages.
- **High-contrast mode:** Dedicated `:root[data-contrast="high"]` with both light and dark variants, thicker input borders (2px).
- **Reduced motion:** Both `@media (prefers-reduced-motion: reduce)` and user-toggle `[data-reduce-motion="true"]` globally zero out animations/transitions.

## 3. Responsive Design (Mobile-First)

### PASS -- Excellent Compliance

- **Breakpoints:** All `@media` queries use `rem` tokens (`48rem`). Zero instances of hardcoded `px` breakpoints found across all CSS files.
- **Viewport units:** Zero uses of `100vh`. All height-based layouts use `dvh` (e.g., `100dvh`, `90dvh`, `70dvh`).
- **Container widths:** Consistently use `width: min(100%, Xrem)` pattern -- `82rem` for page, `36rem` for modals.
- **Touch targets:** All interactive elements meet `min-height: 2.75rem` (44px). On mobile, primary buttons escalate to `min-height: 3rem` (48px).
- **iOS input zoom prevention:** Mobile media query forces `font-size: 1rem` on all input types -- properly covers `text`, `email`, `password`, `tel`, `number`, `date`, `search`, `textarea`, `select`.
- **Safe-area insets:** Comprehensive coverage on bottom nav (`padding-bottom: env(safe-area-inset-bottom)`), modals, drawer header/footer, toast container, sticky action bars, declaration form footer.
- **Modals -> Bottom sheets:** At `max-width: 48rem`, modals get `border-radius: lg lg 0 0`, `width: 100%`, `max-height: 90dvh`, sticky action buttons at bottom with safe-area padding.
- **Table -> Card pattern:** All policing UIs (DOPAMS, Forensic, Social Media) use `data-label` attributes on `<td>` elements with CSS `content: attr(data-label)` for mobile card conversion.
- **Bottom nav:** Fixed bottom bar with `padding-bottom: env(safe-area-inset-bottom)`, hidden on desktop (`min-width: 48rem`), tabs have `min-height: 3.25rem`.
- **Main content offset:** `padding-bottom: calc(3.25rem + env(safe-area-inset-bottom) + var(--space-4))` prevents bottom nav overlap.
- **Toast positioning:** Repositioned above bottom nav on mobile with `bottom: calc(var(--space-4) + 4rem + env(safe-area-inset-bottom))`.
- **Heading typography:** Uses `clamp()` throughout (e.g., `clamp(1.35rem, 3vw, 1.85rem)`).

## 5. Design System Consistency

- **Token usage:** Spacing uses `var(--space-1)` through `var(--space-8)` consistently. Border radius uses `var(--radius-sm/md/lg/xl)`. Shadows use `var(--shadow-xs/sm/md)`.
- **Theme system:** 14 themes defined (light, dark, rolex, nord, dracula, solarized, monokai, catppuccin, gruvbox, onedark, tokyonight, rosepine, ayu, github, sunset) -- all with complete token coverage.

- **Shared design system:** All 5 UI apps share the same `design-system.css` structure (confirmed identical bottom nav, modal, button, toast, skeleton, breadcrumb, and reduced-motion patterns).
- **Button system:** Consistent `ui-btn` with variants (`--primary`, `--secondary`, `--ghost`, `--success`, `--warning`, `--danger`) and sizes (`--sm`, `--md`, `--lg`).

- **Input styling:** All form controls share base styles, with `:hover`, `:active`, `:focus`, `:disabled`, and `.error` states.
- **Status badges:** Comprehensive set (draft, submitted, in-progress, query, approved, rejected, closed) using semantic color tokens.

## 6. Performance

### PASS -- Good Patterns

- **Code splitting:** `React.lazy()` used for all major views across all apps. Citizen: Dashboard, DocumentLocker, Settings, ReportComplaint. Officer: Inbox, TaskDetail, SearchPanel, ComplaintManagement, ServiceConfigView, Settings. Policing UIs: All views lazy-loaded.
- **Suspense:** `<Suspense fallback={<SkeletonBlock ... />}>` wraps lazy components.
- **Error boundary:** All apps wrapped in `<ErrorBoundary>` with Sentry reporting.
- **Debounced search:** Search inputs in policing UIs use 300ms debounce timers.
- **Cache layer:** Citizen app has `readCached`/`writeCached` with TTL, schema versioning, and validation functions. Officer has similar. Telemetry tracking for cache hits/misses.
- **Offline support:** `isOffline` state tracked via `online`/`offline` events, cached data served with stale indicators, mutation buttons disabled when offline.
- **Idle timeout:** `useIdleTimeout` hook with auto-logout, shared across all apps.
- **Skeleton loading:** `SkeletonBlock` components with animation for loading states.

### Findings

| # | Severity | Finding | Location |
|---|----------|---------|----------|
| P1 | LOW | Citizen app eagerly imports 5 views (`AllApplications`, `ServiceCatalog`, `CreateApplication`, `TrackApplication`, `ProfileView`) without `lazy()`. Only Dashboard, DocumentLocker, Settings, and ReportComplaint are lazy-loaded. Moving these to lazy imports could reduce initial bundle size. | `apps/citizen/src/App.tsx:36-41` |
| P2 | INFO | Notification count polling runs every 30s in policing UIs. Consider `requestIdleCallback` or visibility-based pausing to avoid polling when tab is in background. | `apps/dopams-ui/src/App.tsx:143-154`, `apps/forensic-ui/src/App.tsx:116-127`, `apps/social-media-ui/src/App.tsx:119-131` |

## 7. Additional Observations

### Strengths
1. **Offline resilience** is well-implemented with stale data indicators, draft conflict resolution, and cache telemetry.
2. **Form navigation guards** prevent accidental data loss with unsaved-changes confirmation dialogs.
3. **Hash routing** with proper history stack, popstate handling, and deep-link support across all apps.
4. **Error reporting** integrated (Sentry DSN via env var) in all apps.
5. **Keyboard navigation** is thorough -- Escape closes menus/modals/drawers, Tab cycling in focus traps, Enter/Space on custom interactive elements.
6. **Evidence upload** in citizen complaint system includes file type validation, progress tracking, and preview thumbnails.
7. **Voice input with AI parsing** for complaint descriptions -- progressive enhancement pattern.

### Minor Code Quality Notes
- Citizen `App.tsx` has ~60 state variables in a single component. While functional, this could benefit from extraction into custom hooks or context providers for maintainability.
- The 14 themes are a nice feature but add ~400 lines to each `design-system.css`. Consider extracting theme definitions into separate files loaded on demand.
