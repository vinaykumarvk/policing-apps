# Applicant details

## **A) Applicant Identity**

* Applicant Full Name  
* Father/Spouse Name *(where required by service/form)*  
* Applicant Type *(Individual / Firm / Company / Society / Govt / Other)*  
* Relationship to Property *(Owner / Co-owner / Allottee / Attorney / Authorized Representative)*

## **B) Contact Details**

* Mobile Number *(OTP-verified at onboarding, reused in applications)*  
* Alternate Mobile Number *(optional)*  
* Email ID *(optional/mandatory as per service; used for notices & receipts)*  
* Landline / Telephone Number *(optional)*

## **C) Address Details**

* Permanent Address *(Line 1, Line 2, City/Village, District, State, PIN)*  
* Correspondence/Communication Address *(if different from permanent)*  
* Temporary Address *(optional; often same as correspondence)*

## **D) Applicant ID / Proof (recommended for governance portals)**

* Identity Proof Type *(Aadhaar / PAN / Passport / DL / Voter ID / Other)*  
* Identity Proof Number  
* Identity Proof Document Upload *(PDF/JPG)*

## **E) Ownership / Property Linkage (captured in onboarding \+ reused in services)**

* Authority / Office *(PUDA / GMADA / GLADA / etc.)*  
* Property Location *(scheme/city/area)*  
* Sector  
* Property Usage Type *(Residential / Commercial / Institutional / Industrial / Mixed / Other)*  
* Property Type *(Plot / House / Flat / Booth / SCO / SCF / Other)*  
* Property Number / Plot Number  
* Allotment Reference Type *(LOI / Allotment Letter / Other)*  
* Allotment Letter / LOI Number  
* *(Optional but useful)* Property ID / Account ID *(system-generated)*

## **F) Authorization (if applicant is not the owner)**

* Authorized Person Name  
* Authorized Person Mobile  
* Authorized Person Email *(optional)*  
* Authorization Type *(SPA / GPA / Board Resolution / Authorization Letter)*  
* Authorization Document Upload  
* Owner Consent / Declaration Checkbox

## **G) Communication Preferences (recommended)**

* Preferred Communication Mode *(SMS / Email / Both)*  
* Consent for OTP/Notifications *(checkbox)*

## **H) Portal Security / Recovery (onboarding profile)**

* Security Question  
* Security Answer  
* Captcha (system)  
* OTP Verification Status (system)  
* Profile Created Date/Time (system)

## **I) Declarations & Signature**

* Applicant Declaration Acceptance *(checkbox)*  
* Digital Signature / eSign / Upload Signed Form *(depending on process)*  
* Applicant Name (auto-filled under signature)  
* Date (system/application date)

# Property Details

## **A) Core Property Identifiers (Portal “Property Selection” / Linking)**

* Authority / Organization *(PUDA / GMADA / GLADA / PDA / etc.)*  
* Location *(scheme/city/area as per authority master data)* ([GMADA](https://www.gmada.gov.in/sites/default/files/2020-09/Guidelines%20for%20creating%20userid%20%26%20password.pdf?utm_source=chatgpt.com))  
* Sector ([GMADA](https://www.gmada.gov.in/sites/default/files/2020-09/Guidelines%20for%20creating%20userid%20%26%20password.pdf?utm_source=chatgpt.com))  
* Usage Type ([GMADA](https://www.gmada.gov.in/sites/default/files/2020-09/Guidelines%20for%20creating%20userid%20%26%20password.pdf?utm_source=chatgpt.com))  
* Property Type ([GMADA](https://www.gmada.gov.in/sites/default/files/2020-09/Guidelines%20for%20creating%20userid%20%26%20password.pdf?utm_source=chatgpt.com))  
* Property Number / Plot/Unit Number ([GMADA](https://www.gmada.gov.in/sites/default/files/2020-09/Guidelines%20for%20creating%20userid%20%26%20password.pdf?utm_source=chatgpt.com))  
* Unique Property Number (UPN) *(system identifier used as user-id in many cases)* ([GMADA](https://www.gmada.gov.in/sites/default/files/2020-09/Guidelines%20for%20creating%20userid%20%26%20password.pdf?utm_source=chatgpt.com))

## **B) Allotment / Title References (Proof of ownership/allotment)**

* LOI / Allotment Letter Reference Number ([GMADA](https://www.gmada.gov.in/sites/default/files/2020-09/Guidelines%20for%20creating%20userid%20%26%20password.pdf?utm_source=chatgpt.com))  
* Allottee Name (as per allotment record) ([GMADA](https://www.gmada.gov.in/sites/default/files/2020-09/Guidelines%20for%20creating%20userid%20%26%20password.pdf?utm_source=chatgpt.com))  
* *(Common in records even if not always asked upfront)* Allotment Date  
* *(Common)* Conveyance Deed Number & Date *(if executed)*  
* *(Common)* Registry / Mutation Reference *(where applicable)*

## **C) Site / Revenue Identification (Needed for building-related services)**

* Plot Number *(when in an approved sector/scheme)*  
* Revenue particulars (rural/edge areas):  
  * Khasra Number  
  * Village  
  * *(Often also maintained)* Hadbast / Tehsil / District *(recommended)*

---

## **D) Property Address & Geo/Neighbour Context (Site-plan context)**

* Property Address (scheme/sector \+ plot/site no \+ locality) *(recommended)*  
* Site boundaries / plot boundaries (geometry / boundary points)  
* Abutting road(s) \+ road width(s) \+ road name (if any)  
* North direction reference (for plans)  
* Immediate neighbours on all sides (built/vacant)  
* Site levels / plinth levels relative to adjoining road(s)

---

## **E) Property Physical Characteristics (Plot / Unit attributes)**

* Plot/Property Area  
* Plot dimensions (frontage, depth, side lengths) *(typical for scrutiny; recommended)*  
* Corner plot flag *(recommended)*  
* Plot category/type (e.g., booth/SCO/SCF/flat/plot/industrial) *(usually covered by “Property Type”)* ([GMADA](https://www.gmada.gov.in/sites/default/files/2020-09/Guidelines%20for%20creating%20userid%20%26%20password.pdf?utm_source=chatgpt.com))  
* House Number / Site Number (where allotted as “site/house”)

---

## **F) Land Use / Planning Controls (Key for approvals)**

* Permitted land use / occupancy category *(residential/commercial/etc.)* ([GMADA](https://www.gmada.gov.in/sites/default/files/2020-09/Guidelines%20for%20creating%20userid%20%26%20password.pdf?utm_source=chatgpt.com))  
* Ground Coverage (calculated / declared for plan submissions)  
* FAR / N-FAR (calculated / declared)  
* Parking calculations (declared/calculated)  
* Any other plan/scrutiny calculations required by competent authority

---

## **G) Building/Construction Linkages (When the service is building-related)**

* Proposed building outline / footprint (as per plan set)  
* Existing building details (if alteration/addition case) *(recommended)*  
* Sanction/approval order number & date (building plan sanction reference)  
* Description of building (summary fields used on completion/occupancy forms):  
  * Area, Site No, House No, Khasra No, Village

---

## **H) Financial / Ledger / Dues (Common across property-services)**

* Ledger/Account reference (internal) *(recommended)*  
* Current dues / outstanding balance *(recommended)*  
* Payment history / receipts *(recommended)*  
* Installment plan / schedule *(recommended)*  
* Penalty/interest/compounding amounts *(recommended)*

---

## **I) Documents Tied to the Property (Uploads / references)**

* LOI / Allotment Letter (upload/reference) ([GMADA](https://www.gmada.gov.in/sites/default/files/2020-09/Guidelines%20for%20creating%20userid%20%26%20password.pdf?utm_source=chatgpt.com))  
* Conveyance Deed / Sale Deed / Gift Deed / Transfer Order *(as applicable)*  
* Possession letter *(as applicable)*  
* Site plan / building drawings / supporting calculations  
* Any other authority-required supporting documents (NOC, affidavits, etc.) *(service-dependent)*
